# 提示词模板库

根据任务类型选择合适的模板，填充 `{变量}` 后生成最终提示词。

---

## 通用模板

适用于大多数代码修改任务。

```
你正在项目 {project_path} 中工作。

## 任务
{task_description}

## 约束
{constraints}
- 不要删除已有的注释代码，除非明确要求
- 保持现有代码风格一致性

## 完成后输出
请输出修改摘要:
- 【修改文件】: 列出所有修改/新增/删除的文件路径
- 【关键变更】: 每个文件的核心改动（1-2 句话）
- 【风险提示】: 任何需要注意的问题
```

## 功能添加模板

适用于新增功能、新增模块、新增 API 等场景。

```
你正在项目 {project_path} 中工作。

## 任务
添加以下功能: {feature_description}

## 功能需求
{requirements}

## 技术约束
{constraints}
- 遵循项目现有的架构模式和目录结构
- 新增文件应与现有文件风格一致
- 如有需要，更新相关的导入和注册

## 完成后输出
请输出:
- 【新增文件】: 新创建的文件及其用途
- 【修改文件】: 已有文件的变更说明
- 【使用方式】: 如何调用/启用新功能
- 【依赖变更】: 是否新增了外部依赖
```

---

## Bug 修复模板

适用于定位和修复已知问题。

```
你正在项目 {project_path} 中工作。

## 问题描述
{bug_description}

## 复现步骤
{reproduction_steps}

## 期望行为
{expected_behavior}

## 排查方向
{hints}

## 约束
- 修复应尽量小范围，避免大面积重构
- 如果根本原因不在预期位置，说明你的发现
{constraints}

## 完成后输出
请输出:
- 【根本原因】: 问题的根本原因分析
- 【修复方案】: 采用的修复策略
- 【修改文件】: 修改的文件和具体改动
- 【验证建议】: 如何验证修复有效
```

---

## 代码重构模板

适用于代码整理、性能优化、结构调整。

```
你正在项目 {project_path} 中工作。

## 重构目标
{refactoring_goal}

## 涉及范围
{scope}

## 约束
- 重构不应改变外部行为（功能等价）
- 保持向后兼容
{constraints}

## 完成后输出
请输出:
- 【重构内容】: 做了什么结构性调整
- 【修改文件】: 变更的文件列表
- 【改进点】: 重构带来的具体改善
- 【破坏性变更】: 是否有任何 API/接口变更
```

## 调试/分析模板

适用于不确定问题所在，需要 Claude Code 先分析再行动的场景。

```
你正在项目 {project_path} 中工作。

## 问题现象
{symptom_description}

## 已知信息
{known_info}

## 请执行以下步骤
1. 分析相关代码，定位问题根源
2. 如果能确定原因，直接修复
3. 如果不确定，列出可能的原因和排查建议

## 约束
{constraints}

## 完成后输出
请输出:
- 【分析结果】: 问题的根源分析
- 【已执行操作】: 做了哪些修复（如果有）
- 【待确认项】: 需要用户进一步确认的问题
```

---

## 只读分析模板

适用于只需要分析代码但不修改的场景。使用时应将 allowedTools 设为 "Read,Glob,Grep"。

```
你正在项目 {project_path} 中工作。

## 分析目标
{analysis_goal}

## 关注点
{focus_areas}

## 请输出
- 【架构概览】: 相关模块的架构和数据流
- 【代码质量】: 发现的问题或改进机会
- 【建议】: 具体的改进建议和优先级
```

## 变量说明

| 变量 | 说明 | 来源 |
|------|------|------|
| `{project_path}` | 项目根目录绝对路径 | 配置文件或用户指定 |
| `{task_description}` | 任务描述 | 从用户消息提取 |
| `{feature_description}` | 功能描述 | 从用户消息提取 |
| `{requirements}` | 功能需求细节 | 从用户消息提取 |
| `{bug_description}` | Bug 描述 | 从用户消息提取 |
| `{reproduction_steps}` | 复现步骤 | 从用户消息提取，可选 |
| `{expected_behavior}` | 期望行为 | 从用户消息提取 |
| `{hints}` | 排查线索 | 从用户消息提取，可选 |
| `{constraints}` | 额外约束 | 从用户消息提取，可为空 |
| `{refactoring_goal}` | 重构目标 | 从用户消息提取 |
| `{scope}` | 涉及范围 | 从用户消息提取 |
| `{symptom_description}` | 问题现象 | 从用户消息提取 |
| `{known_info}` | 已知信息 | 从用户消息提取 |
| `{analysis_goal}` | 分析目标 | 从用户消息提取 |
| `{focus_areas}` | 关注重点 | 从用户消息提取 |

## 模板选择指南

| 用户意图特征 | 推荐模板 |
|-------------|----------|
| "加个 xxx 功能" / "实现 xxx" | 功能添加模板 |
| "xxx 有 bug" / "xxx 不工作" / "修复 xxx" | Bug 修复模板 |
| "重构 xxx" / "优化 xxx" / "整理 xxx 代码" | 代码重构模板 |
| "看看 xxx" / "分析 xxx" / "xxx 怎么实现的" | 只读分析模板 |
| "xxx 报错了" / "不知道哪出问题" | 调试/分析模板 |
| 其他所有代码修改 | 通用模板 |
