## 需要你补充的截图

### A. OpenClaw 视觉 Hook

要什么样的截图：

- 一张横向截图，最好能同时体现“热度”和“风险讨论”。
- 最稳妥的做法：
  1. GitHub repo 首页或趋势页，能看到 star 数。
  2. 再叠一张媒体报道或安全分析文章封面。
- 如果只能选一张：
  优先选 GitHub repo 首页，因为它最容易一眼说明“这东西很火”。

不建议：

- 纯文章长截图。
- 只有文字没有任何产品界面的画面。
- 分辨率太低、看不清标题的截图。

### B. 故事 1：误改上游代码

要什么样的截图：

- 一张 `git diff` 或 PR diff 截图，明确出现 `nanobot/` 路径变更。
- 如果你有 guardrail 拦截画面，再加一张测试失败或 hook 拦截截图。
- 最理想组合：
  1. “误改发生了”的 diff
  2. “后来被约束体系拦住了”的 guardrail 画面

### C. 故事 2：改完不收尾

要什么样的截图：

- 一组连续截图，而不是单张静态图。
- 最理想是三连：
  1. 后台任务完成通知
  2. auto_continue 把结果注回原会话
  3. agent 继续执行后续动作

如果你只能给两张：

- 一张“完成通知”
- 一张“自动续跑后的新回复/新动作”

### D. 故事 3：改完看不到

要什么样的截图：

- 一张“代码已经改了，但页面没变化”的对比图。
- 一张“自动 rebuild 后页面更新”的对比图。
- 若有 console 里的版本检测或 rebuild 按钮状态，也可以补一张。

## 需要你补充的视频

### 1-2 分钟 Demo 录屏

推荐镜头脚本：

1. 在聊天界面或控制入口发起一个明确任务。
2. 切到后台任务或 console，看到任务开始运行。
3. 展示任务完成通知。
4. 展示 auto_continue 触发后 agent 继续推进。
5. 如果能接上前端自动 rebuild 或 lifecycle 状态变化，效果最好。
6. 最后回到人类确认/放行，强调“半闭环而不是全自动闭环”。

拍摄建议：

- 分辨率 1440p 或 1080p 即可。
- 尽量保持一个主窗口，不要频繁切很多应用。
- 鼠标移动慢一点，方便后续必要时拆成截图。
